# 勤怠一覧画面 使い方ガイド

本ドキュメントでは、「勤怠管理（勤怠一覧）」画面の操作方法や仕様について解説します。
本システムでは、ログインしているアカウントの権限（管理者 / 一般従業員）によって行える操作が異なります。

---

## 1. 勤怠記録の修正方法

打刻漏れ（出勤または退勤の押し忘れ）や押し間違いがあった場合、以下の手順でデータを修正できます。

### 【一般従業員】修正申請を行う
一般従業員は、自身の過去の勤怠データを直接書き換えることはできません。管理者に「修正申請」を提出し、承認されることで反映されます。

1. 日別勤怠詳細の一覧から、修正したい日の行にある **「修正申請」** ボタンをクリックします。
   - 1日に複数回の打刻（中抜けなど）がある場合は、修正したい特定の打刻レコードの「修正申請」をクリックします。
2. 表示されるモーダルで、以下の情報を入力します。
   - **出勤希望時刻** / **退勤希望時刻**: 修正後の正しい時刻を指定します。
     - 出勤のみ、または退勤のみの修正も可能です。
     - 打刻ミスで余分に記録されたデータを消したい場合は、**「クリア」** ボタンで空欄にすることができます。
   - **申請理由 (必須)**: 修正が必要になった理由（例: 「カード忘れのため手動申請」「退勤押し忘れ」など）を詳しく入力します。
3. **「申請する」** ボタンをクリックすると、管理者に申請が送信されます。

> [!NOTE]
> - 申請中は、対象日に `承認待ち` バッジが表示され、管理者が承認または却下するまで追加の修正申請は行えません。
> - 申請内容（変更後の勤務時間の増減など）は、申請前にプレビューで確認できます。

### 【管理者】直接追加・編集・削除を行う
管理者は、申請ワークフローを経由せずに、直接勤怠データを追加・修正・削除できます。

- **直接修正・削除**:
  1. 日別詳細の対象行にある **「修正」** ボタンをクリックし、正しい時刻を入力して保存します。
  2. 不要なレコードは **「削除」** ボタンでデータベースから直接削除できます。
- **新規追加**:
  - 打刻レコード自体が存在しない日（欠勤日や打刻漏れの日）に新規に勤務を追加したい場合、**「追加」** ボタンをクリックして出勤・退勤時刻を新規登録できます。

> [!WARNING]
> **締め処理（ロック）中の制限**
> 対象年月が「締め済（ロック）」状態になっている場合、管理者であっても直接編集や申請の承認を行うことはできません。修正を行う場合は、先に締め処理を解除する必要があります。

> [!IMPORTANT]
> **時間帯が重複しています（ATTENDANCE_OVERLAP）と出る場合**
> 同一日に既に登録されている他の勤務時間帯と一部でも重なる時間帯で新規追加や修正、申請を行おうとすると、重複登録防止の安全機能によりエラーが発生して保存できません。先に既存の重複するレコードの時刻を調整するか、不要なレコードを削除してから再度操作を行ってください。

## 2. 勤怠記録の「履歴」の意味

各勤怠データの横にある **「履歴」** ボタンをクリックすると、そのレコードの「変更履歴（監査ログ）」を確認できます。

### 履歴に記録される情報
- **変更日時** と **操作を実行したユーザー**（申請者および承認した管理者、または直接修正した管理者）
- **変更前** および **変更後** の出勤・退勤時刻
- **申請理由**（従業員が入力したもの）
- **承認・却下コメント**（管理者が入力したもの）

### なぜ履歴が必要なのか？
勤務データは給与計算の根拠となる重要な情報です。改ざんや誤入力を防ぎ、いつ・誰が・どういう理由でデータを修正したかを完全に可視化（監査トレール）することで、健全で透明性の高い勤怠管理を実現しています。

## 3. 打刻元（Source）の意味

勤怠一覧に表示される「打刻元」は、その勤怠データがどのような方法で記録されたかを示しています。

| 表示される打刻元 | 意味・記録方法 |
| :--- | :--- |
| **NFC (PaSoRi)** | 打刻端末のICカードリーダー（PaSoRi）にFeliCaカードをかざして打刻された正規の記録。 |
| **Web打刻 (ID入力)** | ブラウザの打刻画面から、手動で従業員ID（メールアドレス等）を入力して打刻された記録。 |
| **自己申告** | 一般従業員が「修正申請」を行い、管理者がそれを承認して反映された記録。 |
| **管理者修正** | 管理者が管理画面から直接追加、あるいは修正した記録。 |
| **自動補完** | 出勤打刻はあるが退勤打刻がないまま日を跨いだ場合などに、システムスケジュール等によって自動的に退勤処理された記録。 |

## 4. その他、画面で使い方に困りそうなこと

### Q. カレンダーのところどころに挟まっている「週次集計」行は何ですか？
日曜日、または月の最終日の行の下に、その週（月曜日から日曜日まで）の「勤務日数」と「勤務時間の合計」が自動で計算されて挿入されます。週ごとの勤務状況の目安などを把握するためにご活用ください。
※勤務時間の表示形式は、Excel等と共通の `[h]:mm` 形式（例: `8時間15分` の場合は `8:15`、`162時間30分` の場合は `162:30`）となっております。

### Q. 【管理者向け】「月を締める（ロック）」とは何ですか？
月末になり、その月の勤務データが確定した段階で、管理者が「月を締める」を実行します。
- ロックを行うと、その月の勤怠データは一切編集できなくなります（従業員の修正申請、キャンセル、管理者の直接編集すべてがブロックされます）。
- 給与計算中に過去のデータが誤って変更されるのを防ぐための重要な操作です。
- 締め処理を実行した際、まだ承認待ち（pending）の申請が残っている場合は、**「当月の締め処理が完了したため、システムにより自動的に却下されました。」** という理由で自動的に却下されます。
