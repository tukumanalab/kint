# システム設定 使い方ガイド

本ドキュメントでは、管理者向けに「システム設定」画面の各項目の意味、設定可能な値の制限、およびバックアップや打刻端末の管理方法について解説します。

---

## 1. 一般設定

システム全体の基本的な表示情報を設定します。

### サイト名
- **意味**: システムの正式名称です。ブラウザのタブタイトル、ログイン画面、メイン画面のヘッダー表示、および自動送信される通知メールの送信元名称などに反映されます。
- **制限**: **必須入力**（空欄にはできません）。最大文字数は特に制限されていませんが、画面のレイアウト上、50文字以内を推奨します。

### サイトのサブタイトル
- **意味**: システムの補足的な説明です。ログイン画面やメイン画面の上部に、サイト名の下に補助的に表示されます。
- **制限**: **必須入力**（空欄にはできません）。最大文字数は特に制限されていませんが、100文字以内を推奨します。

### ログイン継続時間
- **意味**: ユーザーがログインした状態（JWTアクセストークン）の有効期限を設定します。
- **制限**: **1〜8760の整数（時間）**。デフォルトは `168` 時間（7日間）です。最大で365日分（8760時間）まで設定できます。
- **適用タイミング**: 設定保存後に新しくログインしたセッション（または発行されたアクセストークン）に対して適用されます。

### Google新規登録の許可
- **意味**: Googleログイン時の新規ユーザー自動登録（アカウント作成）の許可・禁止を切り替えます。
- **制限**: トグルスイッチでON/OFFを切り替えます。
- **適用タイミング**: 設定保存後に試みられるすべてのGoogleログインおよび新規登録処理に対して即時適用されます。無効（OFF）にすると、登録済みのユーザーのみGoogleログインが可能になり、未登録のGoogleアカウントからの新規登録は拒否されます。

## 2. 打刻規則

出退勤の打刻に関する制御および判定ルールを設定します。

### 連続打刻クールダウン
- **意味**: 同じカード（FeliCa IDm）で連続して打刻が行われた際、重複打刻（一度に2回かざしてしまうなど）とみなして無視する時間間隔（秒）です。例えば `60` 秒に設定した場合、最初の打刻から60秒以内に同じカードで打刻を試みても、重複と判定され打刻は記録されません。
- **制限**: **0〜3600の整数（秒）**である必要があります。`0` に設定するとクールダウンは無効化され、即時の連続打刻が許可されます。

### シフト開始前チェックイン許容時間
- **意味**: 予定されているシフト開始時刻の何分前から出勤打刻（チェックイン）を認めるかの設定です。例えば `30` 分に設定した場合、シフト開始時刻の30分以上前での打刻は、システム上でシフトとの紐付けやチェックインが制限（または警告）されます。
- **制限**: **0〜120の整数（分）**である必要があります。

### 打刻結果表示時間
- **意味**: 打刻画面（未ログインの待ち受け画面）で、カードの読み取りに成功した際（「出勤しました」「退勤しました」等）またはエラーが発生した際に、その結果メッセージを画面に表示し続ける秒数です。
- **制限**: **1〜300の整数（秒）**である必要があります。デフォルトは `5` 秒程度を推奨します。あまりに短いとメッセージを読み取れず、長すぎると次の打刻者が待たされることになります。

## 3. シフトカレンダー

外部のシフト管理システム（Googleカレンダーなど）からシフト予定を自動で同期・取得するための設定です。

### iCal 同期 URL
- **意味**: 同期対象のカレンダーが公開している iCal 形式（`.ics`）のインターネットアドレスです。Googleカレンダー等の「非公開 URL (iCal)」などを設定します。
- **制限**: **`http://` または `https://` で始まる正しい URL** を入力してください。未設定（空欄）にすると、カレンダーの同期処理は一切行われません。

### 自動同期時刻
- **意味**: 毎日何時に外部の iCal URL からシフト情報を自動的にインポートするかの設定です。
- **制限**: **「時」と「分」のセレクトボックス**で選択してください。未選択（`--`）にすると、毎日の自動同期処理は実行されなくなります。

### 直ちに同期する
- **意味**: iCal 同期 URL からシフト予定を直ちに手動で取り込みます。自動同期の時間を待たずに今すぐデータを反映させたい場合や、設定した iCal 同期 URL から正しくデータが取得できるか（接続やデータの妥当性）を確認したい場合にクリックします。
- **表示結果**:
  - 同期に成功した場合、「同期が成功しました (追加: X件, 更新: Y件, 削除: Z件, スキップ: W件)」のように処理された件数が緑色のアラートで表示されます。
  - 同期に失敗した場合（URL が正しくない、インターネット接続ができない、URL が未設定など）、失敗理由を含めた赤色のエラーメッセージが表示されます。

## 4. 月次勤怠レポート通知

従業員への自動通知機能を設定します。

### 月末の自動メール通知を有効にする（トグルスイッチ）
- **意味**: 月末の自動メール通知機能のON/OFFを切り替えます。スイッチがOFFの間は、自動通知時刻の設定は無視され、メールは送信されません。スイッチ本体をクリックまたはタップすることで切り替えられます。

### 自動通知時刻
- **意味**: 毎月末日の指定された時刻に、当月のアクティブな全従業員（管理者 `role == 'admin'` を除く、メールアドレスが設定されているユーザー）に対し、当月の勤務実績レポート（勤務日数、勤務時間、当年4月からの総勤務時間）をメールで自動通知します（勤務時間は `[h]:mm` 形式で通知されます）。
- **制限**: スイッチがONのとき、**「時」と「分」のセレクトボックス**で時間（例: `20` 時 `00` 分）を選択してください。トグルがONのときは入力が必須となります。
- **背景処理**: 保存されたタイミングで、バックエンドの定期タスクスケジューラ（APScheduler）が再構成されます。

## 5. 設定のインポート／エクスポート

現在のシステム設定情報をファイルとして保存、または別環境から復元します。

- **設定をエクスポート**: 現在の「一般設定」「打刻規則」「シフトカレンダー」「月次レポート通知」の全設定項目を JSON ファイルとしてダウンロードします。
- **設定をインポート**: エクスポートした JSON ファイルをアップロードして設定を適用します。
  - **変更プレビュー**: ファイル選択後、即座には適用されず、**「変更プレビューダイアログ」**が表示されます。
  - 管理者は「変更前」と「変更後」の差分、およびインポートされない無視されたキーなどを確認した上で、**「適用する」**ボタンを押す必要があります。

## 6. データベースのフルバックアップ

SQLite データベースファイル（`.db`）全体のバックアップおよび復元を行います。この機能は、設定情報だけでなく、登録ユーザー、打刻した出退勤記録、NFCカード登録情報を含む**システム全体の全データ**を対象とします。

- **データベースをダウンロード**: データベース全体をファイルとしてPCにエクスポートします。
- **データベースを復元**: バックアップした `.db` ファイルをアップロードしてシステムを復元します。
- **制限と注意**:
  - データベースの復元を実行すると、**既存のすべてのデータが完全に上書きされ、元に戻せません**。
  - 実行前にブラウザ側で最終確認のポップアップが表示されますので、慎重に操作してください。

## 7. 打刻端末制限と登録 (打刻端末管理)

セキュリティ保護および運用の利便性向上のため、許可された特定の端末（ブラウザ）からのみ打刻ページ（未ログイン待ち受け画面）を開けるように制限する機能です。

### 仕組み
- データベース内には端末情報を一切保存しません。
- 代わりに、管理者が端末を登録する際、バックエンドが署名した長期有効な **Device Punch Token (JWT)** を発行し、ブラウザの `localStorage` に保存します。
- バックエンドは、リクエストヘッダー `X-Punch-Device-Token` で受け取ったトークンの署名検証のみを行い、有効なトークンであるか判定します。

### 端末の登録手順
1. 打刻用にする端末（PCやタブレット）のブラウザで、Kint のシステム設定画面を開きます。
2. 画面最下部の「打刻端末管理」セクションにスクロールします。
3. **「現在の端末を打刻用として登録」** で、識別しやすい **「端末名」**（例: 「1F受付用タブレット」「本社玄関PC」など）を入力します。
4. **「登録」** ボタンをクリックすると、そのブラウザにトークンが保存され、次回から誰でも打刻画面を開くことができるようになります。

### 登録の解除
- **現在の端末の解除**: 「打刻端末管理」で登録済みの状態であれば、**「現在の端末の登録を取り消す」** ボタンが表示され、クリックすると `localStorage` のトークンが削除されます。
